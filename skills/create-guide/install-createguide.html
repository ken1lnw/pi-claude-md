คืออะไร?

ไฟล์ create-guide.md ใน folder นี้คือ Claude Code Slash Command Prompt — Markdown ที่เขียนเป็น instruction ให้ Claude ทำตาม เมื่อ copy ไปวางใน .claude/commands/ แล้วรัน /create-guide ใน Claude Code CLI จะสร้างคู่มือ HTML พร้อมใช้อัตโนมัติ

💡

กลไก Claude Code Slash Command:
ทุกไฟล์ *.md ที่อยู่ใน ~/.claude/commands/ จะกลายเป็นคำสั่ง /ชื่อไฟล์ ที่ใช้ได้ใน Claude Code CLI — Claude อ่าน Markdown นั้นเป็น instruction แล้วรัน tools จริง (Read, Write, Glob, Bash, Playwright) ตามขั้นตอนที่เขียนไว้

ไฟล์ในแพ็กเกจ

CLAUDE-SKILL/ ├── install-guide.md ← /install-guide — ติดตั้ง skill อื่นๆ อัตโนมัติ ├── create-guide-th.md ← prompt version ภาษาไทย (~88KB · 1,574 บรรทัด) ├── create-guide-en.md ← prompt version ภาษาอังกฤษ (~73KB · 1,574 บรรทัด) └── create-guide-usage.html ← ไฟล์นี้ — คู่มือติดตั้ง + วิธีใช้

create-guide-th.md และ create-guide-en.md คือ Prompt ไฟล์เดียวกัน แต่ภาษา instruction ต่างกัน — ภาษาของ prompt ไม่ใช่ภาษาของ HTML output (HTML output ควบคุมด้วย flag th / en / th+en แยกต่างหาก)

🔑

ภาษา Prompt vs ภาษา HTML output — ต่างกันอย่างไร?

ภาษา Prompt = ภาษาที่ Claude อ่านเป็น instruction — ไม่มีผลกับ HTML output เลย
ภาษา HTML output = ควบคุมด้วย flag th / en / th+en ที่พิมพ์ตอนใช้ command

สามารถใช้ prompt ภาษาไทย (/create-guide-th) แล้วสร้าง HTML ภาษาอังกฤษ (en) ได้ — และในทางกลับกัน

ตัวอย่างผสมภาษา

# Thai prompt → Thai HTML (ค่า default)
/create-guide-th basket-circulation --capture

# Thai prompt → English HTML
/create-guide-th basket-circulation en --capture

# English prompt → English HTML (ค่า default)
/create-guide-en basket-circulation --capture

# English prompt → Bilingual HTML (Thai + English)
/create-guide-en basket-circulation th+en --capture

เปลี่ยน default ภายหลัง

รัน /install-guide create-guide ใหม่ → เลือกภาษาอื่น → ไฟล์ create-guide.md จะถูกเขียนทับ (TH และ EN แยกยังอยู่ครบ)

/install-guide create-guide   # ถาม default ใหม่ ไม่กระทบ create-guide-th / create-guide-en

create-guide.md ทำงานอย่างไร?

.claude/commands/create-guide.md คือ Prompt-as-Script — Markdown file ที่เขียนเป็น instruction step-by-step ให้ Claude ปฏิบัติตามโดยใช้ tools จริงที่มีใน Claude Code

⚙️

กลไกจริง: เมื่อพิมพ์ /create-guide basket-circulation --capture ใน Claude Code — Claude Code inject เนื้อหา create-guide.md เป็น system instruction แทนที่ $ARGUMENTS ด้วย basket-circulation --capture แล้ว Claude เริ่มรัน Step 0 → Step 4 ตามลำดับ โดยใช้ Read, Write, Glob, Bash, PowerShell, Playwright MCP tools โดยอัตโนมัติ ไม่ต้องเขียน code เพิ่ม

📦

ต้องการ Playwright MCP: flag --capture ใช้ Playwright ถ่ายรูปหน้าจอ ต้องมี Playwright MCP configured ใน Claude Code — ถ้าไม่มี ใช้ได้แค่สร้าง HTML จากรูปที่วางไว้เอง

Workflow ตามสถานการณ์

--capture (ครั้งแรก)

Login → ถามแผน dialog → Playwright screenshot PC+Mobile → อ่าน 4 sources → สร้าง HTML

ไม่มี --capture

ข้าม Login+Playwright → ใช้รูปใน images/ ที่มีอยู่ → อ่าน sources ที่ขาด → สร้าง HTML

--html-only (เร็วสุด)

ข้ามทุกอย่าง → ใช้ guide.config.json cache ทั้งหมด → สร้าง HTML ทันที

--lite (ประหยัด token)

Capture แต่ข้ามการถามแผน + ข้าม extra captures + ข้าม vision analysis

⚠️

CSS Bug ที่ HTML output ระวัง: ใช้ hidden attribute ควบคุม mobile view ไม่ใช่ CSS class — เพราะ .device-mobile{display:none} ใน CSS จะชนะ v.style.display='' ใน JS ทำให้ toggle PC/Mobile ไม่ทำงาน

Capture Flags

--capture
Playwright เข้าระบบ → screenshot ทุกหน้า PC (1440×900) + Mobile (390×844)
--no-mobile
ใช้กับ --capture — capture เฉพาะ PC ข้าม mobile viewport
--replan
ล้าง extraCaptures เดิม → ถามแผน dialog/ปุ่ม ใหม่ทุกหน้า
--recapture
capture รูปใหม่ทุกรูป ไม่ว่ามีรูปเดิมอยู่แล้วหรือไม่

Skip Flags — ลด Token

--no-plan
ข้ามการถามแผน ใช้ extraCaptures เดิมจาก cache
ประหยัด ~500–1,000 tokens
--overview-only
capture overview PC+Mobile เท่านั้น ข้าม dialog/extra ทั้งหมด
ประหยัด ~200–500 tokens
--no-code
ข้ามการอ่าน source code ทั้งหมด ใช้ cache เดิม
ประหยัด ~2,000–5,000 tokens
--no-vision
ข้าม vision analysis รูป ใช้ filename pattern เป็น type แทน
ประหยัด ~800–2,000 tokens/รูป
--html-only
ข้ามทุกอย่าง สร้าง HTML จาก cache เท่านั้น (ต้องมี cache ก่อน)
ประหยัด ~80–95%
--lite
Shorthand ของ --no-plan --overview-only --no-vision พร้อมกัน
ประหยัด ~60–80%

Flagล้าง Cache อะไรใช้เมื่อ
--refresh-metaCLAUDE.md metaแก้ URL หรือ project config

--refresh-codesource code ทุก slugแก้ code component

--freshทั้งหมดสร้างใหม่ตั้งแต่ต้น

/create-guide-th <moduleName> --html-only            # เร็วสุด — จาก cache ล้วน
/create-guide-en <moduleName> --html-only            # English version จาก cache
/create-guide-th <moduleName>                        # อ่าน source ใหม่ด้วย

เพิ่มรูปเองแล้วสร้าง HTML

/create-guide-th <moduleName>                        # รูปชื่อใหม่ → auto-pick
/create-guide-th <moduleName> --refresh-images       # วางรูปทับชื่อเดิม

ประหยัด Token

/create-guide-th <moduleName> --capture --lite       # capture + ข้าม plan+vision
/create-guide-en <moduleName> --capture --no-vision  # English prompt, capture แต่ไม่ vision

Reset ทั้งหมด

/create-guide-th <moduleName> --fresh --capture      # ล้าง cache + สร้างใหม่ทั้งหมด

ความเสี่ยงจาก Screenshot (inherent risk):
